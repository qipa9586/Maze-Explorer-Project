## v1.0 — 核心数据结构与算法

### Cell — 单个格子（v1.0）

```c
typedef struct {
    bool top, right, bottom, left;   // 四面墙，true=有墙挡住
    bool visited;                    // DFS生成和BFS寻路共用
    bool isPath;                     // true=在BFS最短路径上
    bool explored;                   // true=BFS探索过（v1.5加）
    bool playerPath;                 // true=玩家走过（v1.7加）
} Cell;
```

### Maze — 迷宫整体（各版本逐步追加字段）

```c
typedef struct {
    int rows, cols, cellSize;        // 行列数+动态格子尺寸（v2.0改为动态）
    int offsetX, offsetY;            // 居中偏移（v2.0从宏改为字段）
    Cell **grid;                     // 二维格子数组
    Position *path;                  // BFS回溯路径坐标
    int pathLen;                     // 路径长度
    MazeState state;                 // 状态机（v1.5加）
    StackNode *genStack;             // DFS动画栈（v1.5加）
    Queue *bfsQueue;                 // BFS动画队列（v1.5加）
    int **prevRow, **prevCol;        // BFS回溯数组（v1.5加）
    int playerRow, playerCol;        // 玩家位置（v1.7加）
    int playerStep;                  // 计步（v1.7加）
    bool instantMode;                // 一键/动画模式（v2.0加）
    MenuButton menuButton[3];        // 主菜单按钮（v2.0加）
    MenuButton sizeButton[3];        // 难度选择按钮（v2.0加）
    bool confirmReturn, confirmQuit; // 确认弹窗（v2.0加）
    SpeedButton speedButton[4];      // 速度档位按钮（v2.1加）
    int animSpeed;                   // 每帧步数（v2.1加）
    MenuButton saveButton;           // 存档按钮（v2.1加）
    double startTime;                // 计时起点（v2.1加）
    double bestTime[3];              // 最佳纪录（v2.1加）
} Maze;
```

### 辅助结构

- **Position**：`{int col, row}` — 坐标（v1.0）
- **StackNode**：`{Position pos, StackNode *next}` — 链栈节点，哑头设计（v1.0）
- **QueueNode**：`{int row, col, prevRow, prevCol, *next}` — 链队节点（v1.0）
- **Queue**：`{QueueNode *front, *rear}` — 链队带尾指针 O(1) enqueue（v1.0）
- **MenuButton**：`{Rectangle bounds, const char *label}` — 纯手绘按钮（v2.0）
- **SpeedButton**：`{Rectangle bounds, int speed}` — 速度档位按钮（v2.1）

## 算法要点（各版本累积）

### 一、随机化 DFS 生成 — v1.0

```
方向数组：
  dr[4] = {-1,  1,  0,  0}
  dc[4] = { 0,  0, -1,  1}

显式栈（链表实现，哑头）：
  栈顶始终是"当前正在探索的格子"
  初始 push(0, 0)，标记 grid[0][0].visited = true
```

#### 主循环逻辑

```
(0, 0) 标记 visited，push 入栈

while (栈非空):
    current = peek 栈顶

    // 一：收集未访问邻居
    for i = 0..3:
        nr = current.row + dr[i]
        nc = current.col + dc[i]
        如果没越界且 grid[nr][nc].visited == false:
            unvisited 加入 i

    // 二：死胡同回溯
    if unvisited 为空:
        pop 栈顶; continue

    // 三：Fisher-Yates 洗牌随机选
    // 四：对称拆墙
    // 五：标记 visited，push 邻居
```

#### 关键提醒

- **Fisher-Yates 洗牌**：`srand(time(NULL))` 在 `main` 开头调一次
- **拆墙要对称**：当前格的右墙拆了，邻居的左墙也得拆
- **peek 和 pop 的区别**：peek 只看不删，pop 是死胡同才删（回溯）
- **栈空 = 生成完毕**
- **`nextRow`/`nextCol` 是循环最后一次的值**：洗牌后必须用 `dRows[chosen]`/`dCols[chosen]` 重新算

### 二、BFS 寻路 — v1.0（动画扩展 v1.5，双模式 v2.0）

**数据结构**：链队（`front`+`rear` 双指针），FIFO。`QueueNode` 含 row/col + prevRow/prevCol 用于回溯。

**流程**：
1. 重置全部 visited = false，起点 (0,0) 入队标记 visited
2. 偏移数组复用 `dRows/dCols`，与 DFS 生成共享
3. while 循环：dequeue 当前节点，若是终点 → break 进回溯。否则四个方向循环：
   - 越界检查 + 墙打通检查（当前格的对应方向 `== false`）+ 邻居未 visited
   - 三个条件都过 → 记录 `prevRow[nextR][nextC] = curr.row`、标记 visited、入队
4. 回溯：从终点沿 prev 倒回起点，每步设 `isPath = true`，同时存入 `maze->path[]` 数组

**为什么 BFS 保证最短路径**：无权图每步代价 = 1，BFS 层层扩散，第 k 层的格子距离起点恰为 k 步。谁先碰到终点谁最短——因为同一层所有格子距起点一样远，先到的层数最少。

**BFS vs DFS 生成 vs BFS 寻路**：生成只管 visited（挖通），寻路要同时看 visited + 墙是否打通（中间没墙挡才能走过去）。这是 BFS 寻路和 DFS 生成判断条件的最大区别。

**注意**：BFS 的 prev 数组需要在整个搜索期间持久化（不能随出队 free），所以用二维 `int **prevRow/prevCol` 而非 QueueNode 里的字段。

### 三、GUI 渲染 — v1.0 基础（菜单 v2.0，工具栏 v2.1）

**帧循环骨架**：`InitWindow → while(!WindowShouldClose()) { BeginDrawing; ...; EndDrawing; } → CloseWindow`

**居中公式**：`(总空间 - 总内容) / 2`。迷宫用 `offsetX/Y`，按钮组用 `startY`，同一条公式。

**画布分层**（从底到顶）：
1. `ClearBackground` — 清屏
2. `DrawRectangle` 格子底色（DARKGRAY）
3. 条件覆盖：`explored` → 深蓝紫，`isPath` → GOLD 填充
4. `DrawCircle` 起点(GREEN)/终点(RED) 小圆点
5. `DrawLine` 四面墙（WALL_COLOR 浅灰）
6. `DrawLineEx` BFS 最短路径金线（加粗 3px）
7. `DrawRectangle` 玩家橙色方块（PLAYING 状态）
8. UI 文字层：按键提示 / YOU WIN / 确认弹窗

**坐标系**：raylib 原点在窗口左上角，x 右、y 下。`(row, col)` 是逻辑坐标，`(x, y)` 是像素坐标。转换：`x = offsetX + col * cellSize`，`y = offsetY + row * cellSize`。

**自定义按钮**：`Rectangle` + `CheckCollisionPointRec` 判鼠标悬停变色 + `IsMouseButtonPressed` 判点击。无外部 UI 库，纯手绘。

**帧计数器调速**：`static int cooldown; if (cooldown++ % N == 0) { action; }`。玩家移动用 N=5（12步/秒），动画生成用 `for(i=0;i<maze->animSpeed;i++)`。详细 raylib API 参见 `raylib-函数速查手册.md`。

## v1.5 计划 — 算法可视化动画（已完成）

### 核心思路

v1.0 的 while 循环一口气跑到底 → v1.5 拆成 `stepGenerate()` / `stepSolve()`，每帧只执行一次循环体。帧循环跑 `BeginDrawing → drawMaze → EndDrawing`，每帧看到最新状态。

### 三个关键决策

**1. 栈/队列从局部变全局**：v1.0 函数局部变量出函数就销毁 → v1.5 挂 `Maze->genStack` / `Maze->bfsQueue`，跨帧存活。动画版用持久栈（`stepGenerate`），一键版保留局部栈（`generateRandomizedMaze`），两套不冲突。

**2. 状态机驱动**：用 `MazeState` 枚举 + `main.c` 里 `if/else` 分岔，替代散落 bool 变量。状态流转：IDLE → GENERATING → GENERATED → SOLVING → SOLVED。

**3. 每帧只跑一步**：`stepGenerate` 的 while 改成 `if (isEmpty) return true`，只跑一次迭代体。`stepSolve` 同理，每帧 dequeue 一个节点。加 `for(i=0; i<N; i++)` 包层实现可调速度。

### stepGenerate 实现

```c
bool stepGenerate(Maze *maze) {
    if (isStackEmpty(maze->genStack)) return true;  // 完成
    Position curr = peek(maze->genStack);
    // 收集未访问邻居 → 洗牌 → 拆墙 → push → return false
}
```

`return true` = 生成完毕，调用方切 `GENERATED`。`return false` = 只走了一步，下帧继续。

### stepSolve 实现

```c
bool stepSolve(Maze *maze) {
    if (isQueueEmpty(maze->bfsQueue)) return true;
    QueueNode curr; dequeue(maze->bfsQueue, &curr);
    if (curr 是终点) return true;
    // 四个方向：墙打通 + 未访问 → 记录 prev → 标记 visited+explored → enqueue
    return false;
}
```

BFS 探索过的格子设 `explored = true`，gui.c 用深蓝紫色渲染，形成"波前扩散"视觉。

### 改动面一览

| 文件 | 改动 |
|------|------|
| maze.h | 加 genStack、bfsQueue、explored、state、prevRow/Col 字段；stepGenerate/stepSolve 声明 |
| maze.c | createMaze 初始化 state=IDLE，genStack/bfsQueue=NULL，explored=false |
| generate.c | 保留原 generateRandomizedMaze（一锤子）；新增 stepGenerate（逐帧） |
| solver.c | 保留原 mazeSolver（一锤子）；新增 stepSolve（逐帧） |
| gui.c | explored 格子深蓝紫渲染；PLAYING 玩家方块；SOLVING 状态下画波前 |
| main.c | 状态机 if/else 驱动；R 键重新初始化栈；Space 键初始化队列切 SOLVING |

### 已踩坑

- **把 `pop(stack, ...)` 写成 `pop(maze->genStack, ...)`**：generateRandomizedMaze 局部栈被误改死循环，要命
- **prevRow/Col 初始化放 stepSolve 里**：每帧重置把前帧记录清空，移 main.c Space 键做一次
- **stepSolve 每帧 memset prev**：移走后仍残留 for 循环置 -1，全删

## v2.0 — 菜单 + 难度选择（已完成）

### 主菜单

程序启动进入 IDLE 状态。三个手绘按钮纵向排列，居中：

```
Animated Generation  → 初始化 genStack，逐帧 DFS 动画生成
Instant Generation   → generateRandomizedMaze 一锤子 + breakCycles
Player Exploration   → 进入难度选择二级菜单
```

**按钮实现**：`MenuButton` 结构体（bounds + label），raylib 无 UI 库全靠手绘。`CheckCollisionPointRec` 判鼠标悬停变色，`IsMouseButtonPressed` 判点击。居中公式：`startY = WIDTH/2 - (btnH*3 + gap*2)/2`。

### 双模式（instantMode）

`Maze->instantMode` 标志位：按钮 0 设 false（动画版），按钮 1 设 true（一键版）。空格寻路分岔——true 直接 `mazeSolver` 一锤子，false 走 `stepSolve` 逐帧动画。

### 玩家模式（PLAYING）

`IsKeyDown` 按住持续移动（每 5 帧一步），WASD+方向键双控。四面墙检查 + 越界检查。`H` 键静默 BFS 提示路径，再按关闭。走到终点 `WON` 状态显示 YOU WIN + 玩家步数 vs BFS 最短步数。

### 随机破环

DFS 生成完美迷宫后，`breakCycles(maze)` 随机拆约 1-2% 内部墙。只操作 row/col 在 1 到 rows/cols-2 的格子（不碰外墙），对称拆墙。打破"只有一条路"的结构，BFS 仍能找到最短。

### 确认弹窗

`confirmQuit` / `confirmReturn` 两个 bool。ESC 键 → 弹 "Quit? (Y/N)"。PLAYING 下 M 键 → 弹 "Return to difficulty? (Y/N)"。`SetExitKey(0)` 禁用 raylib 默认 ESC 退出。确认框画在 `EndDrawing` 前最顶层。

### 动态迷宫尺寸

`cellSize`/`offsetX`/`offsetY` 从宏改为 Maze 字段。`createMaze(rows, cols)` 自动计算：`cellSize = min(LENGTH/cols, WIDTH/rows)`，offset 用居中公式。三级难度选完调用 `destroyMaze` + `createMaze` 动态重建后按钮数组同步重初始化。

#### drawToolbar 实现引导

**第一步：想想四个按钮和三个信息区分别画在哪**

顶栏全长 1600px，分成三段——左边速度区（300px），中间存档区（100px），右边快捷键提示（1200px 自由空间）。

```
x=0         x=300       x=400                    x=1600
├───────────┼───────────┼──────────────────────────┤
│ [1x][2x]  │  [Save]   │  R:Regen M:Menu Space   │
│  [4x][Max]│           │  H:Hint F12:Shot         │
└──────────────────────────────────────────────────┘
```

每个速度按钮宽约 50px，间距 10px，共四个。存档按钮也 50px。按钮的 y 坐标都在 5-8px 左右（离顶栏上边留一点边距），高度跟 TOOLBAR_H 差不多（约 25-30px）。

**第二步：怎么画当前选中的速度**

遍历四个速度按钮时，判断当前 `animSpeed` 是否等于这个按钮代表的速度值。是的话底色用蓝色（或亮色），不是的话灰底。文字始终黑色。`sprintf` 动态生成标签（"1x"/"2x"/"4x"/"Max"）。

**第三步：不同模式不同按钮**

工具栏内容按状态动态变化：

| 状态 | 显示内容 |
|------|---------|
| IDLE / SELECTING_SIZE | 不显示工具栏 |
| GENERATING（动画模式） | 速度按钮 [1x] [2x] [4x] [Max] + 快捷键提示 |
| GENERATED | 速度按钮（当前档位显示） + 快捷键提示 |
| SOLVING（动画模式） | 速度按钮 + 快捷键提示 |
| SOLVED | 快捷键提示（R:Regen M:Menu） |
| PLAYING | [Save] 按钮 + 快捷键提示（H:Hint M:Menu） |
| WON | 快捷键提示 |

速度按钮只在"动画正在跑"的状态有意义（GENERATING、SOLVING），`instantMode` 下显示但不可点（置灰）。存档按钮只出现在 PLAYING。

**第四步：按钮怎么存数据**

`MenuButton` 复用现有结构体——bounds 存坐标和大小，label 存 "1x" 等。不需要 hovered 字段，drawToolbar 内部即时判断 hovered 就行。

或者新建一个专门的结构体存速度值 + bounds，label 用 `sprintf` 动态生成更灵活：

```c
typedef struct {
    Rectangle bounds;
    int speed;   // 对应的 animSpeed 值
} SpeedButton;
```

**第四步：drawToolbar 放在哪个文件**

放 `gui.c`，`drawMaze` 前面调（或在 `drawMaze` 内部最开始调）。确保在 `BeginDrawing` 之后、迷宫格子之前调用。

**第五步：鼠标点击怎么处理**

`main.c` 里主循环的鼠标点击检测，在 `IDLE` / `SELECTING_SIZE` 判断后面加 toolbar 按钮的点击判断。遍历 `speedBtns[4]`，如果点中了某个按钮，就把 `maze->animSpeed` 设为它的 `speed` 值。存档按钮同理。

**第六步：显示哪些按钮取决于状态**

不是所有状态都显示工具栏。`IDLE` 和 `SELECTING_SIZE` 不显示——它们有自己的菜单界面。只在 `GENERATING` / `GENERATED` / `SOLVING` / `SOLVED` / `PLAYING` / `WON` 显示。速度快键只在动画模式（GENERATING、SOLVING）有意义。存档只在 PLAYING 有意义。这些都可以用 if 判断隐藏或置灰。

### v2.1.2 — 计时 + 最佳纪录 ✅

进 PLAYING 记 `maze->startTime = GetTime()`，到终点算耗时。`bestTime[3]` 存各难度最佳。WON 界面显示秒数 + "New Best!"。

### v2.1.3 — 鼠标拖拽控制 ✅

按住鼠标左键拖拽移动玩家。横向拖多走左右、纵向拖多走上下。`IsMouseButtonPressed` 记起点，`IsMouseButtonReleased` 算偏移判方向。与 WASD 并列可用。鼠标悬停玩家方块变手指光标（`MOUSE_CURSOR_POINTING_HAND`）。

### v2.1.4 — 存档功能

三档难度各一存档槽。`fwrite`/`fread` 二进制存：尺寸 + 墙数据 + 玩家位置 + 步数 + 耗时。进入时检测存档弹 "Continue or New?"。
