## 核心数据结构

### Cell — 单个格子

```c
typedef struct {
    bool top;       // 上墙：true = 存在（不通），false = 打通
    bool right;     // 右墙
    bool bottom;    // 下墙
    bool left;      // 左墙
    bool visited;   // DFS 生成 + BFS 寻路共用（可重置）
} Cell;
```

- 初始状态：四面墙全部 `true`
- DFS 打通相邻两格：各自对应方向的墙都设 `false`
- `visited`：生成阶段标记"已被挖通"；寻路前全部重置为 `false`

### Maze — 迷宫整体

```c
typedef struct {
    int rows;       // 行数（高度，格子数）
    int cols;       // 列数（宽度，格子数）
    Cell **grid;    // 动态二维数组，grid[r][c]
} Maze;
```

### BFS 辅助 — 队列节点

```c
typedef struct {
    int row;
    int col;
    int prevRow;    // 回溯用：从哪一格来的
    int prevCol;
} QueueNode;
```

## 算法要点

### 一、随机化 DFS 生成（显式栈版）

#### 数据结构准备

```
方向数组：
  dr[4] = {-1,  1,  0,  0}    // 上、下、左、右 → 行的变化量
  dc[4] = { 0,  0, -1,  1}    // 上、下、左、右 → 列的变化量

显式栈（链表实现，哑头）：
  栈顶始终是"当前正在探索的格子"
  初始 push(0, 0)，标记 grid[0][0].visited = true
```

#### 主循环逻辑（伪代码）

```
(0, 0) 标记 visited，push 入栈

while (栈非空):
    current = peek 栈顶          // 看当前格，不出栈

    // 第一步：收集未访问邻居
    unvisited = []
    for i = 0..3:               // 四个方向
        nr = current.row + dr[i]
        nc = current.col + dc[i]
        如果 (nr, nc) 没越界 且 grid[nr][nc].visited == false:
            unvisited 加入 i    // 记录方向编号

    // 第二步：分支判断
    if unvisited 为空:
        pop 栈顶                 // 死胡同，回溯
        continue

    // 第三步：随机选一个未访问邻居
    用 Fisher-Yates 洗牌打乱 unvisited 的顺序
    取第一个作为 chosen 方向

    // 第四步：拆墙（两格各改一次）
    根据 chosen 方向：
        chosen == 0（上）: grid[r][c].top    = false; grid[nr][nc].bottom = false;
        chosen == 1（下）: grid[r][c].bottom = false; grid[nr][nc].top    = false;
        chosen == 2（左）: grid[r][c].left   = false; grid[nr][nc].right  = false;
        chosen == 3（右）: grid[r][c].right  = false; grid[nr][nc].left   = false;

    // 第五步：进入邻居
    grid[nr][nc].visited = true
    push(nr, nc)
```

#### 手动走一遍（2×2 迷宫，4 格）

```
初始：四格都四面有墙，visited 全 false
栈 = [(0,0)]，grid[0][0].visited = true

=== 第 1 轮 ===
栈顶 (0,0)，邻居：右(0,1) 未访问，下(1,0) 未访问
随机选 → 右
拆墙：(0,0).right=false, (0,1).left=false
push(0,1)，grid[0][1].visited = true
栈 = [(0,0), (0,1)]

=== 第 2 轮 ===
栈顶 (0,1)，邻居：左(0,0) 已访问，下(1,1) 未访问
只有下 → 选中
拆墙：(0,1).bottom=false, (1,1).top=false
push(1,1)，grid[1][1].visited = true
栈 = [(0,0), (0,1), (1,1)]

=== 第 3 轮 ===
栈顶 (1,1)，邻居：上(0,1) 已访问，左(1,0) 未访问
只有左 → 选中
拆墙：(1,1).left=false, (1,0).right=false
push(1,0)，grid[1][0].visited = true
栈 = [(0,0), (0,1), (1,1), (1,0)]

=== 第 4 轮 ===
栈顶 (1,0)，邻居：上(0,0) 已访问，右(1,1) 已访问
无未访问邻居 → pop(1,0)
栈 = [(0,0), (0,1), (1,1)]

接下来每轮栈顶都没有未访问邻居，逐个 pop 直到栈空 → 生成完毕
```

#### 画图理解墙是如何拆的

```
初始：              第1轮（(0,0)拆右墙）：  第2轮（(0,1)拆下墙）：  第3轮（(1,1)拆左墙）：
 ─ ─  ─ ─          ─ ─     ─ ─          ─ ─     ─ ─          ─ ─     ─ ─
│      │           │        │           │        │           │        │
│  起  │     →     │  起 ○  │     →     │  起 ○  │     →     │  起 ○  │
│      │           │        │           │  ○     │           │  ○  ○  │  
 ─ ─  ─ ─          ─ ─  ─  ─ ─          ─ ─  ─  ─ ─          ─ ─  ─  ─ ─
│      │           │        │           │        │           │        │
│      │           │        │           │  ○     │           │  ○  ○  │
│      │           │        │           │  终    │           │  终    │
 ─ ─  ─ ─          ─ ─  ─  ─ ─          ─ ─  ─  ─ ─          ─ ─     ─ ─
```

访问顺序：(0,0) → (0,1) → (1,1) → (1,0)，连通路径为 Z 字形，四周外墙保留。

#### 关键提醒

- **Fisher-Yates 洗牌**：`srand(time(NULL))` 在 `main` 开头调一次即可
- **拆墙要对称**：当前格的右墙拆了，邻居的左墙也得拆——不然走过去是通的，走回来撞墙
- **peek 和 pop 的区别**：peek 只看不删（还有未访问邻居时留着它继续挖），pop 是死胡同才删（回溯）
- **栈空 = 生成完毕**：所有格子都被访问过，每个连通，墙也拆好了
- **`nextRow`/`nextCol` 是循环最后一次的值**：for 循环收集邻居时 `nextRow`/`nextCol` 不断被覆盖，循环结束留在方向 3（右）。洗牌选中 `chosen` 后必须用 `dRows[chosen]`/`dCols[chosen]` 重新算一遍，否则拆的是错位的墙，定向隔壁却去了别人的邻居。

### 二、BFS 寻路

1. 起点 (0, 0)，终点 (rows-1, cols-1)
2. 队列初始 push 起点，标记 visited
3. 循环出队：
   - 若当前 == 终点 → 回溯路径
   - 否则：检查四个方向
     - 方向对应的墙必须为 `false`（没有墙挡着）
     - 邻居未被 visited
     - 满足则入队，记录 `prevRow/prevCol`（从哪来的）
4. BFS 的 visited 只标记"已入队/已探索"，和墙的打通无关

**路径回溯**：终点顺着 `prevRow/prevCol` 一路倒回起点，把路径上的格子标记出来用于渲染。

#### 主逻辑（伪代码）
```c
void mazeSolver(Maze *maze) {
    // 1. 重置 visited
    // 2. 创建队列，起点入队，标记 visited
    // 3. 偏移数组 dRows/dCols
    // 4. 准备 prev 回溯数组（int **prevR, **prevC）
    // 5. while (!isEmpty(q))
    //      dequeue → 检查是否终点
    //      四个方向：墙通 + 没越界 + 未访问 → 记录 prev，入队
    // 6. 从终点顺着 prev 回溯到起点，标记路径
    // 7. 释放 prev 数组
}
```

#### gui.c — 画迷宫

**1. 计算偏移量让迷宫居中**：

每个格子的像素坐标：`pixelX = OFFSET_X + col × CELL_SIZE`，`pixelY = OFFSET_Y + row × CELL_SIZE`

**2. 画单面墙 — `DrawLine`**：

raylib 里画线就是一行调用：

```c
DrawLine(startX, startY, endX, endY, color);
```

**一个格子的四面墙分别怎么画**（设格子左上角坐标 `x, y`）：

```
       DrawLine(x, y,  x+CELL_SIZE, y)     ← top（上边，从左到右）
       DrawLine(x, y+CELL_SIZE,  x+CELL_SIZE, y+CELL_SIZE)  ← bottom
       DrawLine(x, y,  x, y+CELL_SIZE)     ← left（左边，从上到下）
       DrawLine(x+CELL_SIZE, y,  x+CELL_SIZE, y+CELL_SIZE)  ← right
```

**3. `drawMaze` 完整流程**：

```c
#include "raylib.h"

void drawMaze(Maze *maze) {
    for (int row = 0; row < maze->rows; row++) {
        for (int col = 0; col < maze->cols; col++) {

            int x = OFFSET_X + col * CELL_SIZE;  // 格子左上角 x
            int y = OFFSET_Y + row * CELL_SIZE;  // 格子左上角 y

            Cell cell = maze->grid[row][col];

            // 画格子底色（区分起点/终点/普通）
            // 画四面墙（if 有墙则画线）
        }
    }
}
```

**4. 画格子和墙的伪代码**：

```
for 每行:
    for 每列:
        计算 (x, y)

        // 先画格子填充（底色）
        if 当前是起点 (0,0):      DrawRectangle(x, y, CELL_SIZE, CELL_SIZE, GREEN)
        else if 当前是终点:       DrawRectangle(x, y, CELL_SIZE, CELL_SIZE, RED)
        else:                     DrawRectangle(x, y, CELL_SIZE, CELL_SIZE, 深灰)

        // 再画墙（画在格子底色上面）
        if cell.top:    DrawLine(x, y,  x+CELL_SIZE, y,  WALL_COLOR)
        if cell.bottom: DrawLine(x, y+CELL_SIZE,  x+CELL_SIZE, y+CELL_SIZE,  WALL_COLOR)
        if cell.left:   DrawLine(x, y,  x, y+CELL_SIZE,  WALL_COLOR)
        if cell.right:  DrawLine(x+CELL_SIZE, y,  x+CELL_SIZE, y+CELL_SIZE,  WALL_COLOR)
```

**5. raylib 颜色怎么用**：

```c
// raylib 内置颜色直接用名字：
RAYWHITE, LIGHTGRAY, GRAY, DARKGRAY, BLACK
RED, GREEN, BLUE, YELLOW, GOLD, ORANGE, PURPLE

// 自定义颜色（RGBA，每通道 0-255）：
Color wallColor = { 200, 200, 200, 255 };   // 浅灰墙
Color bgColor   = { 30,  30,  30,  255 };   // 深色背景
```

**6. 画布顺序很重要**：

先画的在底层，后画的在上层。顺序应该是：
1. `ClearBackground(背景色)` — 清屏
2. 画所有格子的底色块（`DrawRectangle`）
3. 画所有墙线（`DrawLine`）
4. （后续 BFS 阶段）画路径高亮

如果先画墙再画格子，格子会把墙盖掉。

**7. 起点终点区分**：

- 起点标记：`(0, 0)` — 绿色填充
- 终点标记：`(maze->rows - 1, maze->cols - 1)` — 红色填充

走完一趟 BFS 后再在上面叠一层路径线即可。

**8. 函数需不需要返回值**：

`drawMaze` 只画不计算，返回 `void` 即可。它只管从 `Maze *maze` 读数据、画到屏幕上。

**9. 画 BFS 路径（连线法）**：

回溯完后 `maze->path` 存了路径上每个格子的坐标，`maze->pathLen` 是长度。用格子中心连线画路径：

```c
for (int i = 0; i < maze->pathLen - 1; i++) {
    // 路径上相邻两格的正中央像素坐标
    int x1 = OFFSET_X + maze->path[i].col * CELL_SIZE + CELL_SIZE / 2;
    int y1 = OFFSET_Y + maze->path[i].row * CELL_SIZE + CELL_SIZE / 2;
    int x2 = OFFSET_X + maze->path[i+1].col * CELL_SIZE + CELL_SIZE / 2;
    int y2 = OFFSET_Y + maze->path[i+1].row * CELL_SIZE + CELL_SIZE / 2;
    DrawLine(x1, y1, x2, y2, GOLD);
}
```

每个格子中心 = `偏移 + 第几行/列 × 格子尺寸 + 半个格子（到中心）`。

注意 `path` 是终点→起点反序，遍历 `0 → pathLen-1` 就是终点画回起点，视觉效果一样。

连线法不填色、不盖墙，一条金线从起点蜿蜒到终点。
